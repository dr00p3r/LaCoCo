# Benchmark escalado: 3 repos × 5 casos (svelte + prettier + mui) — veredicto de Mejora B a escala

**Fecha:** 2026-07-11 · **Split:** `bench15` (swe-polybench-15) · **Índice:** Jina 768d · **SLM:** `qwen3:4b-instruct`
**Instancias (15):** svelte ×5 (510,728,906,907,1116), prettier ×5 (14400,12930,6604,5025,4667),
mui/material-ui ×5 (11451,11858,12406,13690,13778). · Métrica: EditSiteHit@10 / EditSiteMRR.

## Motivación
Con n=8 svelte no se podía saber si el +0.125 de Mejora B (proxy KIND, rescató svelte-464) era real o
ruido. Este escalado a 3 repos distintos re-corre **base (overfetch=1) / kind (overfetch=3, dim del
vector) / edge (overfetch=3, dim edge-derived de node_metadata)** sobre 15 instancias.

## Resultado — EditSiteHit@10 (base / kind / edge), por-repo y pooled

| estrategia | svelte (5) | prettier (4*) | mui (5†) | **POOLED (14)** |
|---|---|---|---|---|
| hybrid    | 0.800/0.800/0.800 | 0.750/0.750/0.750 | 0.200/0.200/0.200 | **0.571 / 0.571 / 0.571** |
| clcr      | 0.800/0.800/0.800 | 0.500/0.500/0.500 | 0.200/0.200/0.200 | **0.500 / 0.500 / 0.500** |
| consensus | 0.800/0.800/0.800 | 0.750/0.750/0.750 | 0.200/0.200/0.200 | **0.571 / 0.571 / 0.571** |

EditSiteMRR pooled (consensus): base 0.342 / kind 0.342 / **edge 0.345** (Δ despreciable).

\* prettier n=4: una instancia excluida del cómputo. † mui n=5 pero solo 1 con gold resoluble (ver abajo).

## Veredicto: **Mejora B NO mejora el retrieval a escala. Se confirma que el +0.125 era un artefacto.**

- **`base == kind == edge` en EditSiteHit** para las 3 estrategias y los 3 repos. El +0.125 de svelte-n=8
  **no replica**: dependía de una sola tarea (svelte-464), que ni siquiera está en este set svelte.
- Matiz honesto: B **sí reordena** el pool (7/14 tareas cambian su top-5 entre base y kind/edge), pero
  **ninguna tarea flipea miss→hit**: el reordenamiento nunca toca la presencia del edit-site en top-K.
  B está "activa" pero su valor neto sobre el edit-site es **cero**.
- MRR igual de plano (±0.003). Ni el proxy KIND ni la dimensión fiel (edge) mueven la aguja.

**Recomendación:** retirar Mejora B (como C1/HyDE) o dejarla como knob no-op off-por-defecto. La señal
dimensional a nivel de *anclaje* no aporta; la que importa vive en la *expansión de grafo* (consensus ya
la usa vía `RELATION_TO_DIM`+`getIntentWeights`). El norte sigue siendo el **consenso** + baselines.

## Hallazgos del escalado (benchmark)

- **Heterogeneidad por repo (reportar SIEMPRE por-repo):** svelte 0.80 y prettier 0.75 → retrieval
  fuerte; mui 0.20 → aparentemente débil, **pero es un artefacto de traducción del gold**: el gold de
  SWE-PolyBench para mui usa forma `Clase.método` (p. ej. `ListItem.render`), y mui son componentes-
  función de React → LaCoCo indexa `ListItem`, no `ListItem.render`. Solo **1/5** mui (material-ui-11858)
  tiene gold resoluble; los otros 4 son miss por el desajuste, no por fallo de retrieval. En las 10
  instancias con gold limpio (svelte 5 + prettier 4 + mui 1), el retrieval ronda **0.75–0.80**.
- **Fricción de entorno (documentada para reproducir):** el `git clone` completo de repos grandes
  (prettier, mui) es lento/flaky aquí (timeouts 900s, "unexpected disconnect"). Solución que funcionó:
  **shallow single-commit fetch** (`git fetch --depth 1 origin <base_commit>`, ~3-4 min, .git de 2-4 MB)
  + **index sin `npm install`** (ts-morph extrae los nodos del AST aunque los imports no resuelvan; el
  gold node aparece igual). Esto habilita retrieval a escala sin el costo/fallo de instalar deps.
- **Loader generalizado a multi-repo** (`import-swe-polybench.ts`): deriva url/tags/display_name/
  source_roots de `inst.repo`; `deriveSourceRoots` escopa el índice al subárbol tocado
  (`packages/material-ui/src` en mui) desde `changed_files`; flag `--append` para acumular repos.

## Reproducir
```bash
export MD=eval/manifests/swe-polybench-15
for arm in "base 1 kind" "kind 3 kind" "edge 3 edge"; do set -- $arm
  LACOCO_ANN_OVERFETCH=$2 LACOCO_ANN_DIM_SOURCE=$3 \
    npm run eval:retrieval -- --run-id 2026-07-11-bench15-$1 --split bench15 --sanitizer-variant baseline --manifests-dir $MD
  npm run eval:metrics:retrieval -- --run-id 2026-07-11-bench15-$1 --manifests-dir $MD
done
```
